第七步：接入 Cloudflare

Cloudflare 主要是用来做：

  1. CDN 加速（全球访问更快）
  2. 免费 HTTPS / SSL
  3. 防攻击（防 CC、DDoS）
  4. DNS 管理更稳定
  5. 页面缓存（博客打开更快）
  6. 隐藏你的真实源站信息
  7. 后期如果做收费产品，也方便接入 WAF、安全策略等

以下是在 Cloudflare 中添加域名、获取两个 NS 的完整详细步骤：

1.注册 / 登录 Cloudflare

前往 https://dash.cloudflare.com，注册账号或直接登录。免费套餐就够用，个人博客无需付费。

2.添加域名（Add a Site）

  1. 登录后，进入 主控制台（Dashboard）
  2. 点击左侧导航的 “Websites” 或顶部的 “+ Add a domain” 按钮
  3. 在输入框中填入你的顶级域名（如 yourdomain.com），不要带 www 或 https://
  4. 点击 “Continue”

3.选择套餐

Cloudflare 会展示套餐页面：

  • 选择 Free（免费） 套餐即可满足个人博客需求
  • 点击 “Continue” 或 “Get started for free”

4.DNS 记录扫描

Cloudflare 会自动扫描你域名现有的 DNS 记录，并将它们导入到 DNS Zone 中。

  • 扫描完成后，页面会列出识别到的 DNS 记录，先不用管，后续配置 GitHub Pages 时再来添加/修改
  • 点击 “Continue”

5.获取两个 NS（Nameservers）⭐ 关键步骤

当你将域名添加为主 DNS 配置（Full Setup）时，Cloudflare 会自动为你的 Zone 分配两个标准 Nameserver，格式为 <名称>.ns.cloudflare.com。

此时页面会显示类似：

每个账号分配到的 NS 名称不同，完整复制这两个值，后续要填到你的域名注册商后台。

6.在域名注册商处替换 NS

步骤5结束后cloudflare页面会处在一个轮询的状态，会去查上面生成的两个NS有没有被挂在域名上，所以这里要先执行步骤八目的是让两个新的NS挂在域名上。
也可以通过工具 https://dnschecker.org/#NS 查看最新的解析状态

7.等待激活

域名注册商最多需要 24 小时来处理 NS 更新。Cloudflare 检测到 NS 生效后会发邮件通知你域名已激活。

激活后在 Cloudflare Dashboard 中，该域名状态会从 “Pending” 变为 “Active”（绿色）。

第十一步（高级）：GitHub Actions 自动部署

以后：

不用再手动：

详细步骤：

1. 创建 GitHub Personal Access Token (PAT)

  1. 打开 GitHub → 右上角头像 → Settings
  2. 左侧菜单最底部 → Developer settings
  3. Personal access tokens → Tokens (classic)
  4. 点击 Generate new token (classic)
  5. 填写：
    Note: hexo-deploy
    Expiration: 选 No expiration 或 1 年
    Scopes: 勾选 repo（完整 repo 权限）
  6. 点击 Generate token
  7. 立刻复制保存，离开页面后无法再看到

2. 把 Token 添加到源码仓库的 Secrets

进入 myblog 仓库（源码仓库）：

  1. Settings → Secrets and variables → Actions
  2. 点击 New repository secret
  3. 填写：
    Name: GH_TOKEN
    Secret: 粘贴刚才复制的 token
  4. 点击 Add secret

3. 确认 _config.yml 部署配置

确保 _config.yml 末尾的 deploy 配置是 HTTPS 格式（不是 SSH）：

deploy:
type: git
repo: https://github.com/duanjianmin/duanjianmin.github.io.git
branch: main

4. 把源码推送到 GitHub

cd d:/work/workspace/myblog
git init
git add .
git commit -m “feat: add GitHub Actions auto deploy”
git remote add origin https://github.com/duanjianmin/myblog.git
git push -u origin main
验证
Push 成功后，打开 myblog 仓库 → Actions 标签页，就能看到工作流在运行。绿色 ✓ 表示部署成功。

之后的流程就是：

写文章 → git add . → git commit → git push → 自动构建部署 → 博客上线
