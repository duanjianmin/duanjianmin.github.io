Wan2.2 TI2V 5B 本地运行指南（Windows + ComfyUI + GGUF 量化版）

适用环境：Windows 11，NVIDIA RTX 5080（16GB VRAM），Python 3.10

本指南使用 ComfyUI + GGUF 量化模型方案，无需 24GB 显存，16GB 可流畅运行。
249帧（约10秒）视频约需 9分钟推理时间。

硬件要求

GPU NVIDIA 16GB VRAM RTX 5080 16GB

CUDA 12.4+ 12.8

常见问题排查

Q1：Cannot find a working triton installation

原因：WanVideoTorchCompileSettings 节点启用了 inductor 后端，Triton 在 Windows 上不可用。

解决：在工作流中找到 WanVideoTorchCompileSettings 节点，右键 → Bypass（或将节点 mode 设为 4）。本指南提供的 JSON 已预先禁用该节点。

Q2：EasyCache: Using cache device: cpu

这是正常行为。EasyCache 将注意力缓存暂存到 CPU 节省显存，需要时再搬回 GPU。

Q3：图片被裁切（上下/左右截断）

原因：ImageResizeKJv2 节点的缩放模式设为 crop，会裁切图片以填满目标分辨率。

解决：将缩放模式改为 pad（加黑边）或 fit（等比缩放）。同时确认分辨率方向与图片方向一致。

Q4：pip install 网络断线（IncompleteRead）

解决：

  1. 优先使用阿里云镜像：-i https://mirrors.aliyun.com/pypi/simple/
  2. 对大文件（>50MB）使用 Windows BITS 断点续传：

Q5：git clone 中途断开（SSL_ERROR_SYSCALL）

解决：使用浅克隆 + 镜像代理：

Q6：CUDA Out of Memory（显存不足）

解决方案（按优先级）：

  1. 减少帧数（先用 49 帧测试）
  2. 确认使用 --lowvram 参数启动 ComfyUI
  3. 关闭其他占用显存的程序（浏览器、游戏等）
  4. 改用更小的 GGUF 量化版本（Q4_K_M 约 3.2GB）
